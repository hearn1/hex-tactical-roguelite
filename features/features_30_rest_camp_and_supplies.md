# Feature 30 - Rest, Camp, and Supplies (Future)

## Goal
Deepen camp nodes with limited recovery choices and optional supply-style resources.

## DnD-Feel Intent
Make rests feel like party management moments instead of a single heal button.

## Source Docs
- `MAP_AND_NODES.md`
- `PROGRESSION_AND_REWARDS.md`
- `GAME_DESIGN.md`
- `SCOPE.md`

## Skeleton Scope
- Add camp choices such as short rest, train, repair gear, brew potion, or prepare for next combat.
- Optionally add a simple supplies resource if approved in planning.
- Display recovery amounts and tradeoffs before confirmation.
- Log camp outcomes.

## Out of Scope
- Full long rest rules.
- Exhaustion subsystem.
- Detailed ration tracking unless explicitly chosen.

## Dependencies
- Existing camp screen.
- Existing inventory and potion systems.

## Acceptance Criteria Draft
- Camp offers multiple meaningful choices.
- Each choice is visible and reversible until confirmed.
- Recovery and buffs apply correctly.
- Tests cover camp effect rules.

## Planning Notes
- Be careful not to make healing too plentiful for longer runs.
- Decide whether supplies are worth the added state.
